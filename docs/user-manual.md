## 总览

### 系统简介

Vibe ServiceDeliver 是网络设备原厂实施项目管理系统，覆盖项目从立项到结项的全生命周期，并贯穿设备资产从出厂到在网运行的完整状态流转。

### 用户角色与权限

| 角色        | 编码             | 主要职责                                       | 入口                          |
| ----------- | ---------------- | ---------------------------------------------- | ----------------------------- |
| 超级管理员  | SUPER_ADMIN      | 系统配置、用户管理、低代码配置、反馈管理        | PC 管理后台                   |
| 项目经理    | PM               | 项目全生命周期管理、任务派发、交付物审核        | PC 管理后台                   |
| 实施工程师  | ENGINEER         | 接单、现场签到、施工拍照、进度上报              | 移动端 H5 + PC               |
| 代理商管理员 | AGENT_ADMIN      | 接单、指派工程师、提交交付物、工作量确认        | PC 精简版 + 移动端 H5         |
| 代理商工程师 | AGENT_ENGINEER   | 现场施工、进度上报、交付物提交                 | 移动端 H5                    |
| 客户        | CUSTOMER         | 进度查看、割接审批、验收签核                    | H5 轻量入口                   |

### 多端入口

| 终端          | 用途                       | 访问方式                              |
| ------------- | -------------------------- | ------------------------------------- |
| PC 管理后台   | 全功能管理                 | 浏览器访问 `/`                         |
| 移动端 H5     | 工程师现场作业             | 浏览器或企微/飞书 App 内打开          |
| 客户端 H5     | 客户进度查看与签核         | 短信/邮件链接                         |
| 代理商端      | 代理商任务管理             | PC 精简版或 H5                         |

## 第一章 超级管理员手册

### 1.1 登录

1. 浏览器访问系统地址（如 `https://vibe.example.com`）
2. 输入账号（手机号或工号）与密码
3. 点击「登录」按钮
4. 首次登录建议修改默认密码并绑定手机号

> **安全提示**：超级管理员账号请使用强密码（≥ 10 位，含大小写字母 + 数字 + 符号），并定期更换。

### 1.2 用户与权限管理

#### 1.2.1 创建用户

1. 进入「系统管理 > 用户管理」
2. 点击「新增用户」按钮
3. 填写用户信息（姓名、手机号、邮箱、所属部门、岗位）
4. 分配角色（可多选）
5. 设置初始密码或选择「发送短信通知用户设置密码」
6. 点击「保存」

#### 1.2.2 角色权限分配

1. 进入「系统管理 > 角色权限」
2. 选择角色，点击「编辑」
3. 在「菜单权限」中勾选该角色可见的菜单
4. 在「按钮权限」中勾选该角色可执行的操作
5. 在「数据权限」中选择数据范围（全部 / 本部门 / 本人 / 自定义）
6. 点击「保存」

#### 1.2.3 数据权限配置

| 数据范围     | 说明                                       |
| ------------ | ------------------------------------------ |
| 全部         | 该角色可查看所有数据                        |
| 本部门       | 仅可查看本部门数据                          |
| 本人         | 仅可查看自己创建或负责的数据                |
| 自定义       | 按部门/区域/客户自定义可见范围              |

### 1.3 组织架构管理

#### 1.3.1 部门管理

1. 进入「系统管理 > 组织架构 > 部门管理」
2. 点击「新增部门」，填写部门名称、上级部门、负责人
3. 支持拖拽调整部门层级

#### 1.3.2 岗位管理

1. 进入「系统管理 > 组织架构 > 岗位管理」
2. 点击「新增岗位」，填写岗位名称、岗位编码、职级

### 1.4 数据字典管理

1. 进入「系统管理 > 数据字典」
2. 字典类型：项目类型、产品线、区域、设备类别、任务类型等
3. 点击「新增字典类型」→ 输入类型名称与编码
4. 在类型下「新增字典数据」→ 输入键值、排序、是否启用

### 1.5 系统配置

#### 1.5.1 通知模板

1. 进入「系统管理 > 系统配置 > 通知模板」
2. 新增模板：填写模板编码（如 `TASK_ASSIGNED`）、标题、内容（支持变量替换）、渠道
3. 启用模板

#### 1.5.2 项目模板

1. 进入「项目管理 > 项目模板」
2. 新增模板：填写名称、适用项目类型、产品线
3. 配置阶段：勘察 / 设计 / 实施 / 验收 / 移交
4. 每个阶段下配置标准任务清单
5. 保存模板

### 1.6 低代码配置（仅 SUPER_ADMIN）

> 详见 [需求总览 - 系统管理模块](./requirement-overview.md#510-系统管理模块功能清单)。

#### 1.6.1 表单配置

1. 进入「系统管理 > 低代码配置 > 表单配置」
2. 点击「新建」
3. 在设计器中：
   - **左侧字段库**：拖拽所需字段到中间画布（输入框/下拉/日期/开关/级联/富文本/文件等）
   - **右侧属性面板**：编辑字段属性（label/required/默认值/校验规则/宽度）
4. 点击「预览」实时查看表单渲染效果
5. 点击「保存」生成 JSON Schema 并持久化

#### 1.6.2 列表配置

1. 进入「系统管理 > 低代码配置 > 列表配置」
2. 点击「新建」
3. 配置列定义、筛选器、操作按钮
4. 配置数据源 API（如 `/api/v1/customers`）
5. 保存

#### 1.6.3 模板库管理

1. 进入「系统管理 > 低代码配置 > 模板库」
2. 点击「新建模板」→ 填写名称与编码
3. 支持操作：复制、导入导出（JSON）、实例化（基于模板一键创建新配置）

#### 1.6.4 运行时渲染

访问 `/lowcode/runtime/{bizType}/{bizId}` 可看到低代码渲染的管理界面，效果与手工开发一致。

### 1.7 反馈管理

1. 进入「系统管理 > 反馈管理」
2. 查看用户提交的反馈（功能建议 / Bug / 咨询）
3. 处理反馈：
   - 查看反馈详情与截图
   - 修改状态：待处理 → 处理中 → 已解决 → 已关闭
   - 填写处理说明
4. 状态变更自动通知提交人

### 1.8 日志管理

#### 1.8.1 操作日志

1. 进入「系统管理 > 操作日志」
2. 按模块、操作类型、操作人、时间范围筛选
3. 查看操作详情（前后值对比）

#### 1.8.2 登录日志

1. 进入「系统管理 > 登录日志」
2. 查看登录记录（IP、设备、登录结果）
3. 异常登录自动告警

## 第二章 项目经理（PM）手册

### 2.1 登录与工作台

1. 浏览器访问系统地址
2. 输入账号密码登录
3. 进入「我的工作台」：
   - **待办事项**：交付物待审核、割接方案待审批、验收签核待发起等
   - **指标卡片**：在建项目数、风险项目数、本月到期、待派任务
   - **我负责的项目**：进度条概览
   - **我的近期任务**：今日/明日待办
   - **最近动态**：项目操作时间线

### 2.2 项目创建

#### 2.2.1 手动创建项目

1. 进入「项目管理 > 项目列表」
2. 点击「+ 创建项目」按钮
3. 填写项目基本信息：
   - 项目名称（必填）
   - 客户（必填，从客户档案选择）
   - 项目类型（新建/扩容/改造/替换/安全）
   - 产品线（路由/交换/无线/安全/DC）
   - 执行模式（自施 / 代施 / 混合）
   - 优先级（P0/P1/P2/P3）
   - PM（默认当前用户）
   - 区域
   - 合同编号
   - 计划开始与结束日期
   - 项目描述
4. 选择项目模板（可选）：自动生成标准阶段与任务
5. 点击「保存」，项目状态为 `INIT`（立项）

#### 2.2.2 项目规划

1. 进入项目详情页 → 「阶段与任务」Tab
2. 调整阶段：增删改阶段、设置阶段时间范围
3. 任务分解：在阶段下创建子任务，填写任务名、类型、计划时间、执行人/代理商
4. 设置里程碑：在「里程碑」Tab 新增，填写名称、预计日期、交付物要求
5. 设置依赖关系：前置任务完成后才能开始后续任务
6. 完成规划后点击「开始执行」，项目状态 `INIT` → `PLAN` → `EXECUTE`

### 2.3 任务派发

#### 2.3.1 派发给自有工程师（自施）

1. 进入「资源调度 > 任务派发」或项目详情页「阶段与任务」
2. 选择待派任务（状态 `PENDING`）
3. 点击「派发」
4. 选择执行人：
   - 手动选择工程师
   - 或点击「智能推荐」按技能/区域/负荷推荐
5. 设置计划开始与结束日期
6. 点击「确认派发」
7. 任务状态 `PENDING` → `ASSIGNED`
8. 工程师收到飞书/钉钉通知

#### 2.3.2 转包给代理商（代施）

1. 选择待派任务，设置 `execute_mode = AGENT`
2. 点击「转包」
3. 选择代理商公司
4. 填写任务范围与要求
5. 上传附加文档（实施方案、验收标准）
6. 设置截止日期
7. 点击「创建转包任务」
8. 转包任务状态 `PENDING`，代理商收到接单通知

### 2.4 交付物审核

1. 工作台「待办事项」中点击「代理商交付物待您审核」
2. 进入「代理商 > 交付审核」
3. 查看交付物详情：
   - 施工照片（必传）
   - 测试记录（必传）
   - 签收单/签字确认（必传）
   - 配置文件（如适用）
   - 代理商备注
4. 审核操作：
   - **通过**：转包任务状态 `SUBMITTED` → `CONFIRMED`，触发工作量确认流程
   - **退回**：填写退回原因 → 状态 `SUBMITTED` → `RETURNED`，代理商补充后重新提交

### 2.5 割接管理

#### 2.5.1 编制割接方案

1. 进入项目详情页 → 「割接」Tab
2. 点击「新建割接方案」
3. 填写：
   - 割接方案名称
   - 割接时间窗口（开始/结束）
   - 影响范围说明
   - 应急联系人
4. 添加操作步骤：
   - 步骤内容
   - 预估耗时
   - 回退方案（每步）
   - 操作人员分工
5. 保存方案，状态 `DRAFT`

#### 2.5.2 提交审批

1. 割接方案详情页 → 点击「提交内部审批」
2. 状态 `DRAFT` → `PENDING_INTERNAL_APPROVAL`
3. 技术主管/总监在「交付管理 > 割接管理」审批
4. 内部通过后，点击「发起客户审批」
5. 系统生成客户签核链接（H5），通过短信/邮件发送给客户
6. 状态 `INTERNAL_APPROVED` → `PENDING_CUSTOMER_APPROVAL`

#### 2.5.3 执行割接

1. 客户审批通过后（状态 `CUSTOMER_APPROVED`）
2. 割接时间到达 → 点击「开始执行」
3. 状态 → `EXECUTING`
4. 按步骤逐项执行：
   - 点击「完成」标记步骤完成（记录耗时）
   - 点击「异常」标记步骤异常 + 填写异常描述 → 自动通知 PM 与技术主管
   - 点击「回退」执行回退方案
5. 所有步骤完成 → 点击「完成割接」
6. 状态 → `COMPLETED`
7. 关联设备状态自动更新为 `ONLINE`
8. 填写割接总结与改进

### 2.6 验收管理

#### 2.6.1 创建验收任务

1. 进入项目详情页 → 「验收」Tab
2. 点击「新建验收任务」
3. 选择适用的验收标准（按产品线/项目类型）
4. 填写验收任务名称
5. 上传测试记录：
   - 功能测试结果
   - 性能测试结果
   - 测试截图/证明材料
6. 保存为草稿（状态 `DRAFT`）

#### 2.6.2 提交验收申请

1. 验收任务详情页 → 点击「提交验收申请」
2. 状态 `DRAFT` → `APPLIED`
3. 等待内部技术审核（状态 → `INTERNAL_AUDITED`）

#### 2.6.3 客户签核

1. 内部审核通过后，点击「发起客户签核」
2. 系统生成客户签核链接（H5），通过短信/邮件发送给客户
3. 状态 `INTERNAL_AUDITED` → `CUSTOMER_SIGNING`
4. 客户在线签核：
   - 通过：状态 → `COMPLETED`
   - 有条件通过：状态 → `COMPLETED`，触发遗留问题登记
   - 驳回：状态 → `REJECTED`，PM 整改后重新提交

### 2.7 项目结项

1. 验收完成后，进入项目详情页
2. 点击「结项」
3. 系统校验：
   - 所有验收完成？
   - 所有文档归档？
   - 所有费用结算？
4. 校验通过 → 状态 `ACCEPT` → `CLOSE`
5. 填写项目复盘记录
6. 上传竣工文档
7. 点击「归档」→ 状态 `CLOSE` → `ARCHIVED`

### 2.8 PM 常见操作快捷入口

| 操作               | 路径                                  |
| ------------------ | ------------------------------------- |
| 我的项目           | 项目管理 > 项目列表（按 PM 筛选）    |
| 待派任务           | 资源调度 > 任务派发（按状态筛选）    |
| 待审核交付物       | 代理商 > 交付审核                    |
| 待审批割接方案     | 交付管理 > 割接管理                  |
| 待发起验收         | 验收管理 > 验收任务                  |
| 项目周报           | 报表中心 > 项目报表                  |

## 第三章 实施工程师手册

### 3.1 登录

#### PC 端

1. 浏览器访问系统地址
2. 输入账号密码登录
3. 进入「我的工作台」

#### 移动端

1. 在企微/飞书 App 内打开应用，或浏览器访问 H5 地址
2. 输入账号密码登录
3. 进入「首页」

### 3.2 查看任务

1. 移动端底部 Tab「任务」
2. 查看分配给自己的任务列表：
   - 待开始：状态 `ASSIGNED`
   - 进行中：状态 `IN_PROGRESS`
   - 已完成：状态 `COMPLETED` / `CONFIRMED`
3. 点击任务查看详情：项目、站点、联系人、任务要求、附件

### 3.3 现场签到

1. 移动端任务详情页 → 点击「开始签到」
2. 系统获取 GPS 定位
3. 校验是否在客户现场范围内（默认 500m）
4. 拍照签到（防作弊，自动添加时间+GPS 水印）
5. 签到成功，记录签到时间

### 3.4 施工步骤跟踪

1. 签到后进入「施工步骤」页面
2. 按任务配置的标准步骤列表逐项执行
3. 每步完成后点击「完成 + 📷」：
   - 拍照上传（设备外观、铭牌 SN、全景照片等）
   - 填写备注
4. 步骤自动记录完成时间与耗时
5. 全部步骤完成后点击「确认提交」
6. 工单状态 `IN_PROGRESS` → `COMPLETED`，等待 PM 确认

### 3.5 异常问题上报

1. 施工过程中如遇异常 → 点击「⚠️ 上报异常」
2. 填写：
   - 异常类型（设备故障 / 现场环境 / 客户配合 / 其他）
   - 影响程度（轻微 / 严重 / 阻断）
   - 问题描述
   - 现场照片
3. 点击「提交上报」
4. 系统自动通知 PM 与技术主管
5. 异常处理跟踪：在「我的消息」中查看回复

### 3.6 工时填报

1. 移动端「我的」→「工时填报」
2. 选择项目与任务
3. 填写工时（小时）与工作内容
4. 选择是否出差
5. 提交 → PM 审批

### 3.7 工程师 FAQ

详见 [附录 A](#附录-a常见问题-faq)。

## 第四章 代理商手册

### 4.1 登录

1. PC 端：浏览器访问系统地址 → 输入代理商账号密码登录
2. 移动端 H5：在企微/飞书 App 内打开应用
3. 代理商账号由原厂超级管理员创建，登录后默认 `tenantType=AGENT`
4. 仅可见分配给自己公司的任务

### 4.2 代理商工作台

1. 登录后进入「代理商工作台」
2. 查看任务概况：
   - 待接单（数量）
   - 进行中（数量）
   - 待审核（数量）
3. 任务列表分桶展示：
   - **待接单**：状态 `PENDING`
   - **进行中**：状态 `ACCEPTED` / `IN_PROGRESS` / `RETURNED`
   - **待审核**：状态 `SUBMITTED`

### 4.3 接单与拒绝

#### 4.3.1 接单

1. 在「待接单」列表中选择任务
2. 查看任务详情：
   - 项目名称（隐藏合同/成本等敏感信息）
   - 任务范围与要求
   - 截止日期
   - PM 联系方式
3. 点击「接单」
4. 指派代理商工程师（从本代理商工程师档案选择）
5. 点击「确认接单」
6. 状态 `PENDING` → `ACCEPTED` → `IN_PROGRESS`

#### 4.3.2 拒绝

1. 在「待接单」列表中选择任务
2. 点击「拒绝」
3. 填写拒绝原因
4. 状态 `PENDING` → `REJECTED`（终态，需 PM 重新指派）

### 4.4 提交交付物

1. 在「进行中」列表中选择任务
2. 点击「提交交付物」
3. 填写：
   - 完成情况（已完成数量/总数）
   - **施工照片**（必传，至少 5 张）
   - **测试记录**（必传）
   - **签收单/签字确认**（必传）
   - 配置文件（如适用）
   - 备注
4. 点击「保存草稿」或「提交」
5. 状态 `IN_PROGRESS` → `SUBMITTED`
6. 等待 PM 审核

### 4.5 进度上报

1. 在任务详情页 → 「进度上报」
2. 填写当前进度百分比与说明
3. 上传过程照片
4. PM 实时可见进度

### 4.6 工作量确认

1. 任务审核通过后（状态 `CONFIRMED`），自动生成工作量确认单
2. 进入「代理商 > 结算管理」
3. 查看工作量确认单（状态 `PM_CONFIRMED`）
4. 核对工作量（人天/站点数/设备台数）与金额
5. 点击「确认」→ 状态 `PENDING`（待财务审批）
6. 或点击「申诉」→ 退回 PM 重新确认

### 4.7 查看结算

1. 进入「代理商 > 结算管理」
2. 查看结算单列表：
   - 审批状态：DRAFT / PM_CONFIRMED / PENDING / APPROVED / REJECTED / CLOSED
   - 付款状态：UNPAID / PAYING / PAID
3. 月度对账单自动生成
4. 付款完成后状态 `CLOSED`

### 4.8 代理商权限边界

| 可以                       | 不可以                   |
| -------------------------- | ------------------------ |
| 查看转包给自己的任务详情    | 看到其他代理商的信息     |
| 确认接单 / 申请延期         | 看到项目完整信息         |
| 上报进度                   | 看到客户/合同/成本等敏感数据 |
| 上传交付物                 | 修改项目计划和设备信息   |
| 现场签到（GPS + 拍照）     | 删除任务                 |
| 上报异常问题                | 修改工作量（仅可确认/申诉） |
| 提交工作量确认              |                          |
| 查看自己公司的结算信息      |                          |

## 第五章 客户手册

### 5.1 登录方式

客户无需账号，通过短信/邮件中的链接访问 H5 页面：

1. 收到原厂发送的短信/邮件（含一次性链接）
2. 点击链接打开 H5 页面
3. 输入手机号 → 获取短信验证码 → 登录
4. Token 有效期 2 小时，过期需重新获取

### 5.2 查看项目进度

1. 登录后进入项目进度页
2. 查看内容：
   - 项目整体进度条（百分比）
   - 当前阶段
   - 预计完成日期
   - 各阶段进展（已完成/进行中/待开始）
   - 可下载文档（设计方案/验收报告等）

### 5.3 割接审批

1. 收到原厂发送的割接审批通知短信/邮件
2. 点击链接登录 H5
3. 查看割接方案详情：
   - 割接时间窗口
   - 影响范围说明
   - 操作步骤
   - 应急联系人
4. 审批操作：
   - **同意**：点击「同意」按钮 → 割接方案状态 `CUSTOMER_APPROVED`
   - **驳回**：点击「驳回」→ 填写驳回原因 → 状态 `CUSTOMER_REJECTED`
5. 审批结果通过短信通知原厂 PM

### 5.4 验收签核

1. 收到原厂发送的验收签核通知短信/邮件
2. 点击链接登录 H5
3. 查看验收内容：
   - 验收任务名称
   - 测试结果（功能/性能/冗余切换）
   - 测试截图/证明材料
   - 自动评分
4. 签核操作：
   - **同意签核**：点击「同意签核」→ 状态 `COMPLETED`
   - **有条件通过**：点击「有条件通过」→ 填写遗留问题 → 状态 `COMPLETED`，遗留问题进入整改跟踪
   - **需要整改**：点击「需要整改」→ 填写整改要求 → 状态 `REJECTED`
5. 签核结果通过短信通知原厂 PM

### 5.5 下载文档

1. 在项目进度页底部「可下载文档」区域
2. 查看可下载的文档列表：
   - 设计方案
   - 初验测试报告
   - 竣工文档
   - 配置备份
3. 点击「下载」按钮，浏览器自动下载

## 附录 A：常见问题 FAQ

### A.1 登录与账号

**Q1：忘记密码怎么办？**
A：联系超级管理员重置密码。重置后请尽快登录修改默认密码。

**Q2：账号被锁定怎么办？**
A：连续 5 次输入错误密码将锁定账号 30 分钟，30 分钟后自动解锁。如需立即解锁请联系超级管理员。

**Q3：Token 过期如何处理？**
A：PC 端 Token 有效期 8 小时，移动端 7 天。Token 过期后系统自动跳转登录页，重新登录即可。

**Q4：客户如何获取登录链接？**
A：客户无需注册账号。原厂在割接审批、验收签核等关键节点会自动通过短信/邮件发送一次性 H5 链接，Token 有效期 2 小时。

### A.2 项目管理

**Q5：项目创建后能否修改执行模式？**
A：项目 `INIT` / `PLAN` 状态可修改 `execute_mode`；`EXECUTE` 状态后不可修改（已有任务派发）。

**Q6：项目状态流转错误怎么办？**
A：状态流转需符合状态机规则，详见 [状态机转换矩阵](./state-machine.md)。如需回退状态，请联系 SUPER_ADMIN 在「操作日志」中查找并人工干预。

**Q7：项目可以删除吗？**
A：仅 `INIT` / `PLAN` 状态的项目可删除（逻辑删除）；`EXECUTE` 及之后状态不可删除，只能归档或取消。

### A.3 设备管理

**Q8：设备 SN 录入错误如何修改？**
A：仅 `IN_FACTORY` 状态可修改 SN；其他状态需先退回仓库再修改。所有修改记录在 `device_status_log`。

**Q9：设备状态流转错误如何处理？**
A：状态流转需符合设备状态机，详见 [状态机 - 设备状态机](./state-machine.md#二设备状态机device)。如遇异常状态，联系设备管理员人工干预。

**Q10：备件领用后能否归还？**
A：可以。在「设备资产 > 备件管理」选择已领用备件 → 点击「归还」→ 库存恢复，记录 `spare_part_log`。

### A.4 任务派发与执行

**Q11：工程师退回任务后怎么办？**
A：任务状态回到 `PENDING`，PM 重新派发或转派给其他工程师。退回原因记录在任务详情中。

**Q12：代理商拒绝接单怎么办？**
A：转包任务状态变为 `REJECTED`（终态）。PM 需重新创建转包任务并指派其他代理商，或改为自施模式派发给自有工程师。

**Q13：施工过程中遇到异常如何上报？**
A：移动端任务详情页 → 点击「⚠️ 上报异常」→ 填写异常类型/影响程度/描述/照片 → 提交。系统自动通知 PM 与技术主管。

### A.5 割接与验收

**Q14：割接方案被客户驳回怎么办？**
A：状态变为 `CUSTOMER_REJECTED`，PM 修改方案后回到 `DRAFT` 状态重新提交内部审批。

**Q15：客户长期不签核验收怎么办？**
A：系统默认 7 天未签核自动告警 PM，PM 可电话联系客户或重新发送签核链接。

**Q16：验收遗留问题如何跟踪？**
A：进入「验收管理 > 遗留问题」查看整改责任人、截止日期、整改进度。超期自动告警升级。

### A.6 财务结算

**Q17：代理商对工作量有异议怎么办？**
A：在「代理商 > 结算管理」中点击「申诉」→ 退回 PM 重新确认。PM 与代理商协商后修改工作量重新提交。

**Q18：付款失败如何处理？**
A：付款状态从 `PAYING` 退回 `UNPAID`，财务可重新发起付款。

### A.7 低代码配置

**Q19：低代码配置菜单看不到？**
A：低代码配置仅 `SUPER_ADMIN` 可见。如需配置权限，请联系超级管理员分配 `lowcode:config:*` 权限。

**Q20：低代码渲染的页面与手工开发的页面效果不一致？**
A：低代码渲染器遵循与手工开发相同的视觉规范。如发现差异，请提交反馈（右下角悬浮按钮）并附截图。

### A.8 反馈与支持

**Q21：遇到 Bug 如何反馈？**
A：点击右下角悬浮反馈按钮 → 选择类型「Bug 报告」→ 填写描述与截图 → 提交。管理员将在「系统管理 > 反馈管理」处理。

**Q22：如何重新打开新手教程？**
A：点击右上角 `?` 图标 → 选择「重新查看教程」。
