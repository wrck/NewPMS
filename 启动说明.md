# Vibe 服务管理系统 - 启动说明

## 一、依赖环境

| 依赖 | 版本要求 | 用途 | 安装指南 |
| --- | --- | --- | --- |
| Docker Desktop | 任意版本 | 中间件容器化 (MySQL/Redis/MinIO/RabbitMQ/ES/XXL-JOB) | https://www.docker.com/products/docker-desktop/ |
| JDK | 17 或更高 | 后端 Spring Boot 3.x | https://adoptium.net/temurin/releases/?version=17 |
| Maven | 3.6+ | 后端依赖构建 | https://maven.apache.org/download.cgi |
| Node.js | 18 或更高 | 前端 Vite 5 构建 | https://nodejs.org/ |

> **说明**：脚本会自动检测上述依赖，缺失某项时会跳过对应服务而不强制退出。例如：未装 Docker 时跳过中间件但仍启动前端；未装 JDK 17 时跳过后端。

### 路径覆盖

如果依赖安装在非默认路径，可通过环境变量强制指定：

```bat
:: 永久设置（推荐写入系统环境变量）
setx VIBE_JAVA_HOME "D:\dev\jdk-17.0.9"
setx VIBE_MAVEN_HOME "E:\apache-maven-3.8.6"
```

或临时设置（仅当前会话生效）：

```bat
set VIBE_JAVA_HOME=D:\dev\jdk-17.0.9
set VIBE_MAVEN_HOME=E:\apache-maven-3.8.6
```

默认会自动扫描以下路径：

- JDK: `D:\ja-netfilter`、`D:\dev\jdk`、`C:\Program Files\Java`、`C:\Program Files\Eclipse Adoptium`、`C:\Program Files\Microsoft`、`C:\Program Files\Amazon Corretto`、`C:\Program Files\Zulu`、`C:\Program Files\BellSoft`，以及 `JAVA_HOME` 环境变量和 `PATH` 上的 `java`
- Maven: `E:\apache-maven-3.8.6`、`E:\apache-maven-3.9.6`、`C:\apache-maven`、`C:\Program Files\Apache\Maven`、`D:\dev\maven`，以及 `MAVEN_HOME` 环境变量和 `PATH` 上的 `mvn`

## 二、一键启停脚本

项目根目录提供 4 个 `.bat` 入口，本质上都是对 `scripts/start.ps1` 的封装：

| 脚本 | 作用 |
| --- | --- |
| [start.bat](file:///d:/常规软件/AICoding/Trae/workspace/ServiceDeliver/start.bat) | 主入口；无参数进入交互菜单，传参直接执行对应操作 |
| [stop.bat](file:///d:/常规软件/AICoding/Trae/workspace/ServiceDeliver/stop.bat) | 一键停止全部服务（前端 → 后端 → 中间件 → 清理 PID） |
| [restart.bat](file:///d:/常规软件/AICoding/Trae/workspace/ServiceDeliver/restart.bat) | 重启全部服务 |
| [status.bat](file:///d:/常规软件/AICoding/Trae/workspace/ServiceDeliver/status.bat) | 查看服务运行状态 |

### 命令行用法

```bat
:: 进入交互菜单（推荐首次使用）
start.bat

:: 一键启动全部（中间件 → 后端 → 前端）
start.bat all

:: 单独启动某个服务
start.bat middleware   :: 仅启动 Docker 中间件
start.bat backend      :: 仅启动后端 vibe-server
start.bat frontends    :: 启动全部前端 (web + mobile + portal)
start.bat web          :: 仅启动 vibe-web (PC 管理后台)
start.bat mobile       :: 仅启动 vibe-mobile (工程师 H5)
start.bat portal       :: 仅启动 vibe-portal (客户门户 H5)

:: 停止 / 状态 / 重启
start.bat stop
start.bat status
start.bat restart
```


**永久覆盖**：直接修改 [.env.ports](file:///workspace/.env.ports) 中的对应行。

**优先级**：shell 环境变量 > .env.ports > 各组件内置默认值

## 四、启动顺序与降级策略

`start.bat all` 按以下顺序启动：

```
1. Docker 中间件   ←  等待健康检查通过（最长 180 秒）
2. 后端 vibe-server ←  等待 8080 端口监听（最长 120 秒）
3. 前端 vite dev    ←  等待各端口监听（每个最长 60 秒）
```

降级策略：

- **无 Docker**：跳过中间件，继续启动后端和前端；后端因连不上 MySQL 会启动失败
- **无 JDK 17**：跳过后端，继续启动前端；前端可访问但 API 调用会报 `ECONNREFUSED`
- **无 Node.js**：跳过前端
- **端口被占用**：自动 `taskkill /F /T` 树形终止占用进程（需管理员权限）

## 五、日志位置

启动过程中的日志会写入项目根目录的 `logs/` 文件夹：

```
logs/
├── backend.out.log    后端标准输出
├── backend.err.log    后端错误输出
├── web.out.log        vibe-web 标准输出
├── web.err.log        vibe-web 错误输出
├── mobile.out.log     vibe-mobile 标准输出
├── mobile.err.log     vibe-mobile 错误输出
├── portal.out.log     vibe-portal 标准输出
└── portal.err.log     vibe-portal 错误输出
```

后端 Spring Boot 自身的日志还会写入 `vibe-server/vibe-server-bootstrap/logs/vibe-server.log`。

进程 PID 信息保存在项目根目录的 `.vibe-pids.json` 文件中，`stop.bat` 会读取它来精确终止进程；如果该文件丢失，会回退到按端口扫描终止。

## 六、常见问题排查

### Q1：双击 start.bat 闪退

**A**：以管理员身份右键运行；或在 cmd 中执行 `start.bat` 查看错误信息。脚本会自动尝试 UAC 提权，但如果用户拒绝提权或组策略禁止，需要手动以管理员身份打开 cmd。

### Q2：后端启动失败，提示 JDK 未检测到

**A**：

1. 确认 JDK 17+ 已安装：`java -version`
2. 如果安装在自定义路径，设置环境变量：
   ```bat
   setx VIBE_JAVA_HOME "你的JDK路径"
   ```
3. 重新打开 cmd 窗口（环境变量需要新窗口生效），再次运行 `start.bat`

### Q3：Docker 容器启动失败

**A**：

1. 确认 Docker Desktop 已启动（任务栏托盘有 Docker 图标）
2. 检查端口冲突：3306/6379/9000/9001/5672/15672/9200/9300/8081 必须未被占用
3. 查看具体错误：`docker compose -f docker-compose.yml up`（不加 -d 可看实时日志）
4. 重置 Docker 数据：`docker compose -f docker-compose.yml down -v`（⚠️ 会删除所有数据）

### Q4：前端访问报 `ECONNREFUSED` / API 502

**A**：后端未启动或未就绪。检查：

1. `start.bat status` 看后端状态
2. 查看 `logs/backend.err.log`
3. 确认 MySQL 已就绪（后端启动需要连接数据库）

### Q5：端口被占用，无法启动

**A**：

1. 查看占用进程：`netstat -ano | findstr :8080`
2. 终止进程：`taskkill /F /T /PID <PID>`
3. 或直接运行 `start.bat stop` 清理所有服务

### Q6：首次启动很慢

**A**：首次启动需要：

- 拉取 Docker 镜像（MySQL/Redis/MinIO/RabbitMQ/ES/XXL-JOB，约 2-5 分钟，取决于网速）
- npm install 安装前端依赖（每个前端约 1-2 分钟）
- Maven 下载后端依赖（首次约 3-5 分钟）
- Spring Boot 启动 + Flyway 数据库迁移（约 30-60 秒）

后续启动会快很多（依赖已缓存），通常 30 秒内可全部就绪。

### Q7：管理员权限相关问题

**A**：`start.bat` 和 `stop.bat` 会自动尝试 UAC 提权。如果提权失败：

1. 右键 `start.bat` → 以管理员身份运行
2. 或先打开"管理员: 命令提示符"，再 `cd` 到项目目录运行

## 七、开发工作流建议

### 日常开发

```bat
:: 早上：一键启动
start.bat all

:: 中午：查看状态
start.bat status

:: 晚上：一键停止
start.bat stop
```

### 仅开发前端（后端已部署在远程）

```bat
:: 仅启动 vibe-web，代理 /api 到远程后端
set VITE_API_TARGET=https://api.example.com
start.bat web
```

### 仅开发后端（前端用线上版本）

```bat
:: 仅启动后端 + 中间件
start.bat middleware
start.bat backend
```

### 重启后端（修改代码后）

```bat
:: 在交互菜单中选 r 是重启全部，仅重启后端请：
start.bat stop
start.bat middleware
start.bat backend
```
